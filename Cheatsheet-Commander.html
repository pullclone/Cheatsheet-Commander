🧩 Kate (GUI Optimization)

Kate is a powerful LSP-enabled editor, but many vital tools are unmapped by default. Bridge standard IDE muscle memory with Kate's architecture by assigning these via Ctrl+Alt+, (Configure Shortcuts).

🎯 Phased Learning Strategy

Phase 1 (Days 1-3): Lock the home row (HIAE / TRSCZ). Practice rolling inward.

Phase 2 (Days 4-7): Introduce top row (BYOU / DNGVQ). Note the strong left-hand vowel group (OU).

Phase 3 (Days 8-14): Integrate central punctuation. Practice coding syntax ((); "" !=) for index-finger stretches.

Phase 4 (Days 15+): Shadow-type your code without outputting characters to build structural memory. Expect a temporary productivity dip before rapid acceleration.

⌨️ Ergonomic Philosophy

Home Row Anchors: Left hand rests on H I A E, right hand on T R S C. These 8 keys cover ~70% of standard text.

Centralized Punctuation: Programming heavyweights ((), ;:, !?) are grouped in the center index-finger columns, eliminating right-pinky overload for coding.

Editor Navigation: Standard hjkl navigation will be scattered. Do not remap Vim/Emacs keys. Instead, map your keyboard firmware layers to output Arrow Keys directly on the home row (e.g., holding a thumb key turns T R S C into ← ↓ ↑ →).
